## How the key files relate

**`flake.nix`** is the entry point. It declares all inputs (nixpkgs, deploy-rs, disko, sops-nix) and calls `flakeHelpers.nix` to generate both `nixosConfigurations.pebble` and `deploy.nodes.pebble` from a single function call.

**`flakeHelpers.nix`** provides `mkNixos` and `mkMerge`. `mkNixos` takes a hostname, a nixpkgs reference, and extra modules, then produces both the NixOS system configuration and the deploy-rs node definition. `mkMerge` uses `lib.attrsets.recursiveUpdate` with `foldl'` to combine multiple machine definitions into one flake output. For a single-server setup this is optional but pays off when adding machines later.

**`machines/nixos/vars.nix`** holds shared variables passed to all modules via `specialArgs`:
```nix
{
  domain = "grab-lab.gg";
  serverIP = "192.168.10.50";
  adminEmail = "admin@grab-lab.gg";
  timeZone = "America/New_York";  # adjust
  nasIP = "192.168.10.X";
}
```

**`machines/nixos/pebble/default.nix`** is the machine entry point. It imports `disko.nix`, `hardware.nix`, enables specific services, and sets machine-specific config (hostname, networking).

**`homelab/default.nix`** imports all service modules. Each service module uses `mkEnableOption` so services are opt-in вЂ” the machine config enables only the desired services:
```nix
# homelab/default.nix
{ ... }: {
  imports = [
    ./caddy
    ./pihole
    ./grafana
    ./prometheus
    ./loki
    ./home-assistant
    ./uptime-kuma
    ./homepage
    ./netbird
    ./backup
  ];
}
```

**`machines/nixos/pebble/default.nix`** enables services:
```nix
{ ... }: {
  imports = [ ./disko.nix ./hardware.nix ];
  networking.hostName = "pebble";
  my.services.pihole.enable = true;
  my.services.caddy.enable = true;
  my.services.grafana.enable = true;
  # ...
}
```
