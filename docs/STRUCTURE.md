# STRUCTURE.md — Repository Layout

## Directory structure adapted from notthebee/nix-config for a two-machine homelab (pebble + VPS)

```
homelab/
├── flake.nix                  # Top-level flake: inputs, nixosConfigurations, deploy nodes
├── flake.lock                 # Pinned dependency versions
├── flakeHelpers.nix           # mkNixos, mkMerge helper functions
├── justfile                   # Task runner recipes
├── .sops.yaml                 # sops-nix key management rules
│
├── machines/
│   └── nixos/
│       ├── vars.nix           # Shared variables (domain, IP, email, timezone)
│       ├── _common/           # Base NixOS config shared across all hosts
│       │   ├── default.nix    # Imports all common modules
│       │   ├── nix-settings.nix
│       │   ├── ssh.nix
│       │   ├── users.nix
│       │   └── locale.nix
│       ├── pebble/            # HP ProDesk (homelab server, Stage 1-10)
│       │   ├── default.nix    # Machine entry point: enables services, sets hostname
│       │   ├── disko.nix      # ZFS layout via disko
│       │   └── hardware.nix   # Generated by nixos-generate-config
│       ├── boulder/           # ⚠ NOT YET CREATED (Stage 11, Phase 2)
│       └── vps/               # Hetzner CX22 (NetBird control plane, public IP)
│           ├── default.nix    # VPS entry point: hostname, GRUB, virtio modules, sops, firewall
│           ├── disko.nix      # Simple ext4 layout (no ZFS needed on VPS)
│           ├── netbird-server.nix      # OCI containers: netbird-management, signal, dashboard + native coturn
│           └── caddy.nix      # Native NixOS Caddy: TLS termination + gRPC proxy for NetBird
│           # NOTE: netbird-server.nix (services.netbird.server) is NOT used —
│           #       services.netbird.server is not production-ready as of nixos-25.11
├── homelab/                   # Service modules (one directory per service)
│   ├── default.nix            # Imports all service modules
│   ├── caddy/
│   │   └── default.nix        # Caddy reverse proxy + Cloudflare DNS-01
│   ├── pihole/

│   ├── vaultwarden/
│   │   └── default.nix        # Vaultwarden password manager (Stage 5)
│   ├── grafana/
│   │   └── default.nix
│   ├── prometheus/
│   │   └── default.nix
│   ├── loki/
│   │   └── default.nix
│   ├── home-assistant/
│   │   └── default.nix
│   ├── uptime-kuma/
│   │   └── default.nix
│   ├── homepage/
│   │   └── default.nix
│   ├── netbird/
│   │   └── default.nix        # NetBird client (pebble): services.netbird.clients.wt0
│   ├── kanidm/
│   │   └── default.nix        # Kanidm IdP: services.kanidm + declarative users/groups
│   │                          # NOTE: each service module co-locates its OAuth2 client
│   │                          # definition alongside its service config (not centralized here)
│   ├── mosquitto/
│   │   └── default.nix        # MQTT broker (native, Stage 9a)
│   ├── wyoming/
│   │   └── default.nix        # Voice pipeline: Whisper STT + Piper TTS + OpenWakeWord (native, Stage 9b)
│   ├── matter-server/

│   ├── backup/
│   │   └── default.nix        # Sanoid, syncoid, restic
│   │
│   │   # === boulder (machine 2) services ===
│   ├── immich/
│   │   └── default.nix        # Photo management (Stage 14)
│   ├── jellyfin/
│   │   └── default.nix        # Media server (Stage 15)
│   ├── paperless/
│   │   └── default.nix        # Document management (Stage 13)
│   └── postgresql/
│       └── default.nix        # Shared PostgreSQL for boulder services (Stage 12)
│
├── modules/                   # Reusable utility modules (shared across machines)
│   ├── networking/
│   │   └── default.nix        # my.networking.staticIPv4 custom module
│   └── podman/

│
├── secrets/
│   ├── secrets.yaml           # Homelab secrets (pebble + boulder age keys) — Pi-hole, Grafana, MQTT, Vaultwarden, etc.
│   ├── vps.yaml               # VPS secrets (VPS age key) — NetBird TURN pwd, encryption key, relay secret
│   └── README.md              # How to add/rotate secrets (recovery procedure documented)
│
└── users/
    └── admin/
        └── default.nix        # User config, SSH authorized keys
```

## How the key files relate

**`flake.nix`** is the entry point. It declares all inputs (nixpkgs, deploy-rs, disko, sops-nix) and calls `flakeHelpers.nix` to generate `nixosConfigurations` and `deploy.nodes` for **two machines**: `pebble` (homelab) and `vps` (NetBird control plane). See NIX-PATTERNS.md Pattern 12 for the complete multi-machine flake structure.

**`flakeHelpers.nix`** provides `mkNixos` and `mkMerge`. `mkNixos` takes a hostname, a nixpkgs reference, and extra modules, then produces both the NixOS system configuration and the deploy-rs node definition. `mkMerge` uses `lib.attrsets.recursiveUpdate` with `foldl'` to combine both machine definitions (pebble + vps) into one flake output.

**`.sops.yaml`** must contain age keys for **all four** decryption identities: admin workstation (koksownik), pebble SSH host key, boulder SSH host key, and VPS SSH host key. `secrets/secrets.yaml` is encrypted to pebble + boulder + admin; `secrets/vps.yaml` is encrypted to vps + admin. After provisioning each new machine, run `ssh-keyscan <IP> | ssh-to-age` to get its age key, add it to `.sops.yaml`, and rekey relevant secret files.

**`machines/nixos/vars.nix`** holds shared variables passed to all modules via `specialArgs`:
```nix
{
  domain = "grab-lab.gg";
  serverIP = "192.168.10.50";
  adminEmail = "admin@grab-lab.gg";
  timeZone = "America/New_York";  # adjust
  nasIP = "192.168.10.X";
}
```

**`machines/nixos/pebble/default.nix`** is the machine entry point. It imports `disko.nix`, `hardware.nix`, enables specific services, and sets machine-specific config (hostname, networking).

**`homelab/default.nix`** imports all service modules. Each service module uses `mkEnableOption` so services are opt-in — the machine config enables only the desired services:
```nix
# homelab/default.nix
{ ... }: {
  imports = [
    # === pebble (machine 1) services ===
    ./caddy              # Stage 4
    ./pihole             # Stage 3
    ./vaultwarden        # Stage 5
    ./grafana            # Stage 6
    ./prometheus         # Stage 6
    ./loki               # Stage 6
    ./home-assistant     # Stage 9a — includes ESPHome container config
    ./uptime-kuma        # Stage 9a
    ./homepage           # Stage 8
    ./netbird            # Stage 7b
    ./kanidm             # Stage 7c — OIDC+LDAP IdP; OAuth2 clients co-located in service modules
    ./mosquitto          # Stage 9a
    ./wyoming            # Stage 9b — Whisper + Piper + OpenWakeWord
    ./matter-server      # Stage 9b
    ./backup             # Stage 10

    # === boulder (machine 2) services ===
    ./postgresql         # Stage 12
    ./paperless          # Stage 13
    ./immich             # Stage 14
    ./jellyfin           # Stage 15
  ];
}
```

**Kanidm OAuth2 clients are co-located with service configs:** The `homelab/kanidm/default.nix` module defines only the core IdP (users, groups, server settings). Each service module that uses OIDC adds its own `services.kanidm.provision.systems.oauth2.<name>` block in its own `default.nix`. This keeps auth config co-located with service config — for example, `homelab/grafana/default.nix` contains both the Grafana service config and the Kanidm OAuth2 client for Grafana. See `docs/IDP-STRATEGY.md` and NIX-PATTERNS.md Pattern 21.

**Voice services in a single `wyoming/` module:** All three Wyoming services (faster-whisper, piper, openwakeword) are grouped in one `homelab/wyoming/default.nix` because they share the `services.wyoming.*` namespace, are always deployed together as a voice pipeline, and together require the single `ProcSubset` workaround. Splitting them into separate files would add boilerplate with no benefit at this scale.

**`machines/nixos/pebble/default.nix`** enables services:
```nix
{ ... }: {
  imports = [ ./disko.nix ./hardware.nix ];
  networking.hostName = "pebble";
  my.services.pihole.enable = true;
  my.services.caddy.enable = true;
  my.services.vaultwarden.enable = true;
  my.services.grafana.enable = true;
  my.services.kanidm.enable = true;  # Stage 7c
  # ...
}
```

**`machines/nixos/boulder/default.nix`** enables boulder-specific services:
```nix
{ ... }: {
  imports = [ ./disko.nix ./hardware.nix ];
  networking.hostName = "boulder";
  my.services.postgresql.enable = true;
  my.services.paperless.enable = true;
  my.services.immich.enable = true;
  my.services.jellyfin.enable = true;
  # ...
}
```
